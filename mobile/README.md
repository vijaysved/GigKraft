# GigKraft Mobile

Expo + React Native + TypeScript app (Expo SDK 56, Expo Router file-based
navigation). One app for all roles: routing and screens are gated by the
authenticated user's role.

- `pro` -> ProShell (bottom tabs: Leads, Stats, + Add Kraft, Network, Account)
- `homeowner` -> HomeShell (bottom tabs: Discover, Messages, Emergency,
  Recommend, You)
- `node_manager` -> informational screen pointing to the web admin (this role
  is not served by mobile)

## Setup

```
cp .env.example .env       # adjust EXPO_PUBLIC_API_BASE_URL for your machine
npm install
npm run generate:api       # regenerate API types from the backend OpenAPI spec
```

When testing on an Android emulator, set `EXPO_PUBLIC_API_BASE_URL` to
`http://10.0.2.2:8000`. When testing on a physical device, use your machine's
LAN IP instead of localhost.

## Run

```
npx expo start             # Metro dev server; press a/i/w for android/ios/web
npm run android
npm run ios                # macOS only
npm run web
```

Expo runs on the host for device/simulator access; the compose service is
only for web preview / CI.

## Project structure

```
src/
  app/                 Expo Router routes
    _layout.tsx        Root: fonts + ThemeProvider + AuthProvider + Stack
    index.tsx          Splash / role-based entry; redirects by JWT role
    (auth)/            Auth stack: login (email/password + mock OTP), register
    (pro)/             ProShell bottom tabs (placeholder content until M5)
    (home)/            HomeShell bottom tabs (placeholder content until M5)
    web-admin.tsx      node_manager "use the web admin" screen
  api/
    generated/types.ts Generated by `npm run generate:api` (openapi-typescript)
    client.ts          openapi-fetch client + bearer-token middleware
    auth.ts            Typed wrappers: login, register, refresh, me, OTP
  auth/
    AuthContext.tsx    user/role/status + login/register/otp/logout
    session.ts         access/refresh tokens in expo-secure-store
    jwt.ts             claim decoding + expiry checks (no verification)
  theme/
    tokens.ts          Design tokens from design handoff (spacing, radius, ...)
    themes.ts          4 named themes: gigkraft_light, warm_light,
                       cool_light, gigkraft_dark
    ThemeContext.tsx   scheme/themeName/setTheme/toggle; persisted in
                       AsyncStorage (key `gk.theme`)
  components/          Shared primitives: GkButton, GkCard, GkField,
                       GkSelect, GkBadge, GkAvatar, PhoneScaffold, and
                       shell helpers (RaisedTabButton, PlaceholderScreen,
                       AccountPanel)
  config.ts            EXPO_PUBLIC_* env access
```

## Auth and session

- Access and refresh JWTs are stored in `expo-secure-store` (in-memory
  fallback on web so `expo start --web` previews work).
- On app open the session restores automatically: a valid access token is
  reused, otherwise the refresh token is exchanged for a new access token,
  then `/api/me` populates the user.
- Phone OTP is mocked in Phase 1: the backend returns the code in `dev_code`
  and the login screen displays it.

## Theming

Four themes (3 light + 1 dark) defined as plain StyleSheet token objects --
no styling library. The active theme persists across launches and can be
switched from the Account tab in either shell. Colors follow the Manrope +
Mantine-blue design handoff.

## API types

`npm run generate:api` regenerates `src/api/generated/types.ts` from
`../backend/openapi/openapi.json`. Re-run it whenever the backend API
changes. The generated file is committed so the app builds without the
backend checked out.
