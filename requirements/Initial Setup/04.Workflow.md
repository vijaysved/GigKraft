# GigKraft — Roles & Workflows

> Inventory of product roles, user workflows, target platform (web or mobile), and MVP status.
> Status reflects **Phase 1** (`dev/` local stack, integrations mocked) as of **2026-06-12**.
> See also: `01.Basic Structure.md`, `02.Testing Checklist.md`, `Claude Design/design_handoff_cursor/02_SCREENS.md`.

---

## Status Legend

| Status | Meaning |
|---|---|
| **COMPLETED** | Workflow is implemented for MVP Phase 1 (backend + client UI wired, or backend-only where no UI is required). |
| **NOT APPLICABLE** | Explicitly out of MVP scope per locked product decisions. |
| **PENDING** | In product scope but not yet built, or backend-only with no client UI yet. Target: remaining M5 polish or M6 (Phase 2). |

---

## Roles

| Role | Code | Primary platform | Auth methods | Notes |
|---|---|---|---|---|
| **Handyman Pro** | `pro` | Mobile (Expo / React Native) | Email/password, phone OTP (mock) | One mobile app; lands in **ProShell** after sign-in. |
| **Homeowner** | `homeowner` | Mobile (Expo / React Native) | Email/password, phone OTP (mock) | Same mobile app; lands in **HomeShell** after sign-in. |
| **Node Manager** | `node_manager` | Web (React + Mantine) | Email/password, Google (mock) | Desktop admin console. Mobile shows a redirect to web admin. |

**Shared:** All roles use one `User` model with a `role` field and JWT access/refresh tokens. Realtime is polling-based in MVP (WebSocket is Phase 2+).
