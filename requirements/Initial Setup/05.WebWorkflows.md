# GigKraft — Web Workflows & UI Spec (Review Draft)

> **Purpose:** Specification for a unified **React + Mantine + Vite** web client covering **all three roles** (Pro, Homeowner, Node Manager) and every workflow from `04.Workflow.md`.  
> **Status:** Approved (2026-06-12) — W1 theme gallery **in progress**.  
> **Target display:** 14-inch laptop (see §2).  
> **Brand assets:** Logo card + blueprint wallpaper (provided 2026-06-12).  
> **Related docs:** `01.Basic Structure.md`, `04.Workflow.md`, `02_SCREENS.md`, `01_DESIGN_SYSTEM.md`.

---

## 1. Scope & Decisions (for review)

| Topic | Current MVP (`dev/frontend`) | This spec proposes |
|---|---|---|
| Roles on web | Node Manager only | **All three roles** on web (`pro`, `homeowner`, `node_manager`) |
| Mobile | Pro + Homeowner primary today | **Future:** all roles on mobile too |
| Stack | React 18, Vite, TS, Mantine, React Router | Same; add TanStack Query + shared component library |
| Themes | 4 themes | **5 themes** — gallery at `/themes` (W1) |
| Admin nav | Single `/dashboard` page | Dedicated **AdminShell** sidebar (6 sections) |
| Pro / Home nav | N/A on web | **ProShell** + **HomeShell** with **240px sidebar** + split pane |

**Locked decisions (approved 2026-06-12):**

- **All roles on web and mobile** — web ships first for all three roles; mobile will gain all roles in a future release.
- **Default theme:** `gigkraft_brand` globally for all roles.
- **Navigation:** **Sidebar (240px)** for Pro, Homeowner, and Node Manager — not top nav.
- **Leads / messages:** **Split pane** on desktop (list left, detail right); stack below 1280px.
- **Registration:** **Open** pro and homeowner registration on web.
- One web app at `dev/frontend` — role chosen at login/register; JWT `user.role` routes to the correct shell.
- Polling for live updates in Phase 1 (no WebSocket UI yet).
- Media uploads remain URL strings in Phase 1 (S3 presign in Phase 2).
- Homeowner in-app payments stay **out of scope** (same as mobile MVP).

## 2. Viewport & Layout — 14-inch monitor

### 2.1 Target hardware

| Property | Value | Notes |
|---|---|---|
| Physical | 14" laptop | Primary design target |
| Common resolutions | **1366×768**, 1920×1080 | Design at 1366×768; verify at 1920×1080 |
| Effective chrome | ~1280×680 content | After OS taskbar + browser chrome |
| Min supported | **1280×720** | Below this: show compact nav + horizontal scroll on tables only |
| Max content width | **1440px** | Centered; side margins on large displays |

### 2.2 Layout patterns by role

```
┌─────────────────────────────────────────────────────────────┐
│  PUBLIC (login / register / magic-link review)                │
│  Full-bleed blueprint wallpaper + centered auth card (420px)│
└─────────────────────────────────────────────────────────────┘

┌──────────┬──────────────────────────────────────────────────┐
│ Sidebar  │  Top bar (title · breadcrumbs · theme · user)    │
│ 240px    ├──────────────────────────────────────────────────┤
│ fixed    │  Main content (scroll) — padding md (16px)       │
│          │  Max inner width 1120px for forms & tables         │
└──────────┴──────────────────────────────────────────────────┘
  AdminShell (node_manager) — always sidebar

┌──────────┬──────────────────────────────┬─────────────────────┐
│ Sidebar  │  Main list / feed (flex 1)   │  Detail panel       │
│ 240px    │  min 480px                   │  360–420px fixed    │
│ fixed    │                              │  (split pane)       │
│          │  Top bar: title · theme · user                     │
└──────────┴──────────────────────────────┴─────────────────────┘
  ProShell / HomeShell — sidebar + split pane (stack below 1280px)
```

### 2.3 Density rules (14" friendly)

- **No horizontal page scroll** on primary flows at 1366px.
- Data tables: sticky header, virtualized rows when > 50 items, default page size 25.
- Forms: single column ≤ 560px; two-column only when viewport ≥ 1280px.
- Modals: `size="md"` (560px) default; `lg` (720px) for Kraft editor and chat quote cards.
- Touch targets ≥ 40px height even on web (accessibility + trackpad consistency).

## 4. Brand, Logo, Wallpaper & Five Themes

### 4.1 Asset placement

Copy provided images into the frontend public folder at build time:

| Asset | Source | Target path | Usage |
|---|---|---|---|
| Logo card | User-provided logo PNG | `public/brand/logo-card.png` | Login card, sidebar mark, favicon source |
| Blueprint wallpaper | User-provided wallpaper PNG | `public/brand/wallpaper-blueprint.png` | Public auth background, optional admin sidebar texture |
| Favicon | Derived from logo | `public/favicon.ico` | Browser tab |

`<WallpaperBackground />` — full-viewport layer:

- `background-image: url(/brand/wallpaper-blueprint.png)`
- `background-size: cover; background-position: center`
- Overlay: `linear-gradient(180deg, rgba(10,37,64,0.85) 0%, rgba(10,37,64,0.92) 100%)` so form cards remain readable
- Optional CSS `background-blend-mode: multiply` on dark themes

`<GkLogo />` — renders logo card at heights `32 | 48 | 64 | 96` px; includes wordmark **GIGKRAFT** + script tagline on marketing surfaces only.

### 4.2 Brand color tokens (from logo)

These extend (not replace) tokens in `01_DESIGN_SYSTEM.md`:

| Token | Hex | Role |
|---|---|---|
| `--gk-brand-blue` | `#3498DB` | Logo "G", gradient top, pro accent |
| `--gk-brand-orange` | `#F39C12` | Logo "K", gradient bottom, CTA accent |
| `--gk-brand-navy` | `#0A2540` | Wallpaper base, admin chrome |
| `--gk-brand-gradient` | `linear-gradient(135deg, #3498DB 0%, #7B8FA1 50%, #F39C12 100%)` | Hero bands, primary marketing |
| `--gk-proof-lime` | `#CCFF00` | Verified / proof badges (optional hi-vis) |
| `--gk-script` | `'Dancing Script', cursive` | Tagline only |

Primary **action** color stays Mantine **blue-6** (`#228BE6`) for button consistency across themes; brand orange used for secondary CTAs and homeowner emergency actions.

### 4.3 Five themes

Persist selection in `localStorage` key `gigkraft.theme` (same key as today). All themes share Manrope + mono; only surfaces and accents change.

| # | ID | Label | Scheme | Description |
|---|---|---|---|---|
| 1 | `gigkraft_brand` | **GigKraft Brand** *(default)* | Light | Cream canvas `#FAF8F5`, brand gradient header strip, blue primary, orange secondary buttons |
| 2 | `blueprint` | **Blueprint** | Dark | Navy `#0A2540` canvas, wallpaper texture on shell chrome, blue-orange gradient accents on active nav |
| 3 | `warm_light` | Warm Light | Light | Existing warm neutrals (mobile parity) |
| 4 | `cool_light` | Cool Light | Light | Existing cool/teal tint (homeowner-friendly) |
| 5 | `gigkraft_dark` | GigKraft Dark | Dark | Existing dark surfaces (OLED-friendly) |

**Theme selector UI:** `SegmentedControl` or icon popover in top bar on every shell. Preview swatch shows gradient chip for brand/blueprint themes.

**Per-role default (override on first visit only):**

| Role | Suggested default |
|---|---|
| `node_manager` | `blueprint` |
| `pro` | `gigkraft_brand` |
| `homeowner` | `cool_light` |

User choice always wins after first manual switch.

## 13. Responsive & Accessibility

- **Focus order:** sidebar → main → actions; skip link "Skip to content"
- **Color contrast:** WCAG AA minimum on all themes (verify orange-on-white for small text)
- **Keyboard:** DataTable rows focusable; Enter opens detail
- **Reduced motion:** disable emergency pulse when `prefers-reduced-motion`
