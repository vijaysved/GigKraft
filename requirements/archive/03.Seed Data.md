# seed_data — Local Development Seed Command

## Overview

`seed_data` is a Django management command that populates the local database with a minimal but realistic dataset for development and testing. It is safe to re-run — all objects are created with `get_or_create` so nothing is duplicated.

## Usage

```bash
# From GigKraft/dev/backend/ with .venv active

# Standard run (skips objects that already exist)
python manage.py seed_data

# Wipe all seed objects and re-create from scratch
python manage.py seed_data --reset
```

## Idempotency

Re-running the command without `--reset` prints `Already exists` for each object and makes no changes. This means it is safe to include in CI setup scripts or run after pulling new migrations.

## `--reset` Flag

`--reset` deletes all seed users (matched by email / phone) and the `austin-tx` node, then re-creates everything from scratch. Use this when model fields have changed and you need clean seed data.

```bash
python manage.py seed_data --reset
```
